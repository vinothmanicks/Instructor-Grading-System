CANNAPACEUS

Instructor Grading Program

Online Help Documentation

Cannapaceus is a software tool secondary-level professors and professional instructors. It serves as a total digital replacement for a traditional gradebook and a means of consolidating electronic resources pertaining to student grades and other classroom information.

This help document is broken down into sections pertaining to the functionality of the software and the means by which the user interacts with it. Refer to the table of contents on the left side of the page for help with specific parts of the program.

Login Page

Users will access the Instructor Grading Program through a traditional username/password login interface. When the user first runs the program, he or she will create a database by entering a username and password of their choice, whereupon they will be asked to confirm these details.

This username and password will serve as their login information for all future usage of the software. Passwords must be at least 8 characters in length.


Term View

The home page, also known as Term View, is where the user can view current, past, and future terms as well as lists of all courses in each term.


The Sidebar

The Sidebar is a persistent navigation element present across all views. It provides quick and easy returns to several key parts of the program.


The Commit/Revert Buttons

Two buttons, labled "Commit Changes" and "Revert Changes" are present at the bottom of almost all views. They allow the user to see the effects of potential changes without immediately overwriting existing data. All changes are highlighted in green until the users presses the "Commit Changes" button. The "Revert Changes" button undoes all changes made since the last commit.


How to Add or Edit a Term

After clicking the "Add Term" button on the Term View or clicking the "Edit" icon to the right of the term's name, the user must fill out a form with the term's information, including:

  • Season (Winter, Spring, Summer, Fall)
  • The Academic Year

How to Add or Edit a Course

After creating a term, the user can add a course to the Term by clicking the "Add Course" button underneath the selected term or edit a course's information by clicking the "Edit" icon to the right of the course's name, and filling in the course's information, including:

  • The name of the course
  • The course ID
  • The department that the course falls under
  • OPTIONAL: A grading scale, defined as a percentage added to the students' average grades as a decimal value between 0.0 and 1.0
  • OPTIONAL: A course can be tagged as "Archive" to indicate that the course is completed and that no additional edits to grades or other information can be made

Course View

After selecting a course from Term View, the user is directed to Course View. From Course View, the user can view and edit the list of students in a course, the list of assignments, view course statistics, and view a course grade book.


Student View

To access Student View, the user must first populate a list of students by selecting the "Add Student" button from the student tab in Course View


To add a student, fill in the student's information, including:

  • The student's first name and middle initial, seperated by a comma
  • The student's last name
  • The student's unique institutional ID number
  • The student's institutional email address

Click on the name of student to enter Student View. From Student View, the user can see the student's grade for every assignment in the course, as well as the student's average grade for the course.

Assignments are sorted based on category, but the sorting order may be changed by clicking the corresponding key at the top of the table. The assignment grade can be edited from this view. Each grade has four properties:

  • OVERDUE: The assignment was completed after the assigned due date. The user can make manual grade deductions at their discretion.
  • MISSING: The assignment was not completed at all. Unless the user specifies otherwise, this will result is a grade of zero.
  • DROPPED: The assignment was completed, but is not being counted towards the student's average grade.
  • SUBMITTED: The assignment was completed, regardless of timing, and is being counted towards the student's average grades. A grade value will not appear as part of the student's average until an assignment is labeled as SUBMITTED.

To create or email a grade report for an individual student, use the additional options on the right side of the view.


Assignment View

To access Assignment View, the user must first populate a list of assignment categories applicable to the course. To add an assignment category, select the "Add Assignment" button from the assignment tab in Course View.


To add an assignment category, fill in the following information:

  • The name of the category
  • The weight of the category, defined as a percentage of the student's average grade. Weights across all assignment categories should add up to 1.0 for accurate calculation. Unweighted categories should be given the default value of 1.0.
  • If desired, the program will automatically drop the n lowest grades in a category.

After creating assignment categories, the user can create assignments by selecting the "Add Assignment" button underneath the desired category.

An unweighted "Uncategorized" category is available by default.

To create an assignment, fill in the assignment's information, including:

  • The name of the assignment
  • The maximum score of the assignment (any grade higher than this value indicates the presence of bonus points or extra credit)
  • OPTIONAL: The user can specify an assignment's weight to be a value other than that of it's corresponding category.
  • The assignment category under which the assignment's grades will be weighed. If a category is not selected, the assignment will default to the "Uncategorized" category.
  • The date on which the assignment is due
  • The date on which the assignment is made available to students

Assignment View functions similarly to Student View. The table displays a single assignment grade for each student in the course. Grade values and properties can be edited from this view.

Assigment View also displays statistics for the selected assignment, including mean, median, mode, and standard deviation.

Grade values will not be calculated until an assigment is marked as submitted. To mark an assignment as submitted for all students in a course, select the "Submit all Grades" button.


Grade book view

From Course View, the user can access a course's Grade Book by seleting the "Grade Book" button at the top right. The Grade Book shows every grade in the course, sorted by Student.

The Grade Book also displays the student's average grade. Grade values can be edited from this view.


Additional Course Functions

By expanding the hamburger menu at the top-right corner of Course View, the user can access additional course functions, including:

  • The ability to download and print a comprehensive grade report for the entire course as a .PDF file.
  • The ability to download and print a list of students and all student information not related to grades as a .PDF file.
  • The ability to download and print grade reports for each individual student in the course as a .PDF file.
  • The ability to email individual grade reports to students in the course.
  • The ability to import a grading scale, category list, or assignment list from a previous course. See the next section for more details on this function.
  • The ability to export course details to a reusable .CSV file.

How to use a previous grade scheme

From the "Addtional Course Functions" list, the user can choose to import a grading scheme from another course. This functionality will overwrite any existing grading scale, category list, or assignment list.


Question Bank

By selecting the "QBank" button from the sidebar, the user can maintain a list of questions that can be used to automatically generate printed or timed online assignments.

The user can generate an assignment from selected questions by selecting the "Generate Assignment" button, with the option to instantiate a new assignment or to count as an existing assignment.


To create a question, the user will need to provide the following information:

  • The question text
  • The estimated time to complete the question in minutes
  • The maximum amount of points the question will be worth
  • The question type: Multiple Choice, Fill in the Blank, True/False, Short Answer, or Long Answer
  • The relative difficulty of the question: Easy, Medium, Hard, or Higher Order Thinking Skills
  • The assignment category that an assignment containing this question would be classified under
  • For multiple choice questions, the user will need to create answers, both correct and incorrect, for the questions.

User Settings

The user settings menu contains a field where the user can specify the email address and password of their desired email account to use the program's email functionality.

Future versions of the programs will contain additional personalization options.
